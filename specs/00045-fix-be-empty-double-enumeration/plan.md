# Implementation Plan: Fix BeEmpty Double Enumeration

**Branch**: `00045-fix-be-empty-double-enumeration` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/00045-fix-be-empty-double-enumeration/spec.md`

## Summary

`GenericCollectionAssertions<T>.BeEmpty` currently calls `.Any()` and then `.Count()` on the subject, enumerating it twice. Replace both calls with a single upfront `.Count()` so the method is safe for non-replayable sequences while preserving the existing failure-message format.

**Constraints**: Must not add catch blocks; must not change public API signature or failure-message format  
**Scale/Scope**: Single method change in one file
